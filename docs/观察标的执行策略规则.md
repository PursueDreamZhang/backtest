# 观察标的执行策略规则

## 一、策略定位

这套规则用于把人工给出的方向、个股和 ETF 标的，整理成每日可执行的观察清单。

它不负责自动下单，也不直接替代主观判断，而是每天回答三个问题：

1. 哪些标的已经接近可以行动的位置。
2. 哪些标的还值得继续等。
3. 哪些标的已经走坏，暂时不该再看。

核心思想来自《趋势投资买卖逻辑总结》《首次公开！职业级短线交易策略》《弹簧策略思考1》《弹簧策略思考2》：

- 先有趋势，后有弹簧。
- 只在支撑附近、赔率合适的位置试错。
- 价格比观点重要，跌破证伪点就撤。
- 错的时候亏小，对的时候拿住。

## 二、输入标的池

用户每次给出一个观察主题，可以包含方向、个股和 ETF。

建议输入结构如下：

```json
{
  "direction": "机器人",
  "thesis": "板块可能完成二次探底，观察是否重新转强",
  "symbols": [
    {
      "symbol": "159770",
      "name": "机器人ETF",
      "type": "etf",
      "role": "板块代理"
    },
    {
      "symbol": "300124",
      "name": "汇川技术",
      "type": "stock",
      "role": "中军"
    },
    {
      "symbol": "002747",
      "name": "埃斯顿",
      "type": "stock",
      "role": "弹性标的"
    }
  ]
}
```

第一版只要求 `symbol` 必填，其余字段用于报告展示和人工复盘。

## 三、数据窗口

每次运行时，拉取以下数据：

- 最新可用日线价格。
- 最近半年日线行情，至少需要 120 个交易日。
- 字段包括 `open / high / low / close / volume`。

如果标的数据不足 60 个交易日，不参与正式筛选，只放入 `数据不足` 分组。

## 四、执行模式

策略支持两种执行模式：

1. `intraday`：盘中执行模式。
2. `close_confirmed`：收盘确认模式。

盘中模式适合在交易时间内做观察和预警，帮助快速发现价格接近支撑、跌破支撑或开始转强的标的。

收盘确认模式适合在交易日结束后做正式复盘和次日计划，所有日线信号以收盘价和全天成交量为准。

### 1. 盘中执行模式

盘中执行时，历史日线仍然使用最近半年已完成交易日数据，今日行情使用实时行情源补充。

实时行情字段来自 `data/realtime_quote_source.py`，主要使用：

| 字段 | 用途 |
| --- | --- |
| `price` | 当前最新价，作为今日临时收盘价 |
| `open` | 今日开盘价 |
| `previous_close` | 昨日收盘价 |
| `high` | 今日盘中最高价 |
| `low` | 今日盘中最低价 |
| `volume` | 当前成交量 |
| `amount` | 当前成交额 |
| `bid1_price / ask1_price` | 判断买一卖一附近流动性 |
| `trade_time` | 行情对应交易时间 |
| `quote_date` | 行情日期，用于判断是否为当日行情 |
| `market_session` | 盘中阶段，例如 `continuous` 或 `closed` |
| `is_stale` | 是否为过期行情 |
| `volume_unit` | 成交量单位，策略统一要求为 `share` |
| `fetched_at` | 本地抓取时间 |

盘中会构造一根“今日临时 K 线”：

```text
open = 实时 open
high = 实时 high
low = 实时 low
close = 实时 price
volume = 实时 volume
```

这根临时 K 线可以用于判断价格位置，但不能完全等同于正式日线。

盘中可直接使用的信号：

- 当前价格接近 MA20 / MA40 / 前低支撑。
- 当前价格跌破止损位或关键支撑。
- 当前价格重新站上 MA5。
- 当前价格突破近 5 日高点或近 20 日高点。
- 当前涨跌幅相对昨日收盘价是否转强：`price / previous_close - 1 >= 1%`。
- 当前价格距离止损位是否仍然有合理赔率：`risk_to_stop_pct <= 12%`。

赔率分层规则：

- `risk_to_stop_pct <= 12%`：赔率合理，可保留原有触发或观察分层。
- `12% < risk_to_stop_pct <= 18%`：不再列为正式触发，最多降级到 `重点观察 / 盘中预警`。
- `risk_to_stop_pct > 18%`：位置偏离支撑过远，降级到 `等待回调 / 盘中等待`。

盘中只做预警、不做最终确认的信号：

- 缩量回调。
- 收盘站稳 MA5 / MA20 / MA40。
- 收盘突破平台或前高。
- 放量阳线是否成立。
- 日线级别弹簧是否真正松开。

盘中输出分组使用更谨慎的命名：

| 分组 | 含义 |
| --- | --- |
| `盘中触发` | 实时价已经满足转强或突破条件，但仍需收盘确认 |
| `盘中预警` | 实时价接近支撑或买点区域，等待转强 |
| `盘中失效` | 实时价已经跌破支撑、止损位或证伪点 |
| `盘中等待` | 趋势尚可，但价格距离最近支撑超过 5% |

盘中触发不等于正式买点。报告中必须标记为 `provisional`，提醒这是临时信号。

### 2. 盘中量能进度

盘中不能直接用“今日成交量是否缩量”来判断，因为全天成交还没结束。

第一版使用简化的量能进度：

```text
volume_progress = 当前成交量 / 近20日平均成交量
```

并结合交易时间粗略解释：

| 时间 | 判断方式 |
| --- | --- |
| 10:30 前 | `volume_progress >= 0.40` 视为异常放量，不判断缩量 |
| 10:30 到 13:30 | `volume_progress >= 0.70` 视为放量预警 |
| 13:30 后 | `volume_progress >= 1.00` 视为放量预警，`volume_progress <= 0.75` 视为初步缩量 |
| 收盘后 | 使用全天成交量正式确认 |

盘中量能只用于提示，不单独决定买入。

### 3. 收盘确认模式

收盘确认模式使用完整日线数据重新计算全部指标。

收盘后可以正式确认：

- 今日是否站上 MA5 / MA20 / MA40。
- 今日是否真正突破近 5 日或近 20 日高点。
- 今日成交量是否放大或缩小。
- 今日是否形成有效阳线。
- 今日是否跌破止损位并需要排除。

收盘输出分组仍使用：

- `触发买点`
- `重点观察`
- `等待回调`
- `排除观察`
- `数据不足`

### 4. 收盘确认模式的硬前提

以下规则适用于所有买入模型，是 `close_confirmed` 模式下的统一前提：

1. 如果某个标的缺少请求当天的正式日线数据，则不允许继续按前一交易日数据计算任何买入模型。
2. 这类标的直接进入 `数据不足`，并标记为 `缺少当日收盘数据`。
3. 只有当最新一根日线就是请求当天的正式收盘数据时，才允许继续计算模型 A、模型 B、模型 C、模型 D。

这条规则优先级高于所有具体买点模型。也就是说，哪怕某个标的在前一交易日已经满足某个模型，只要请求当天的正式日线还没拿到，收盘确认模式下也必须直接略过，不能沿用旧信号。

## 五、核心指标

每个标的计算以下指标：

| 指标 | 用途 |
| --- | --- |
| MA5 | 判断短线是否重新转强 |
| MA20 | 判断强趋势中的第一次回档 |
| MA40 | 判断中期成本线和核心弹簧位置 |
| MA120 | 判断是否已经走入更长周期弱势 |
| 20日涨跌幅 | 判断近期动量 |
| 40日涨跌幅 | 判断中期趋势 |
| 60日相对强弱 | 判断是否比同组 ETF 或市场更强 |
| 近20日最高价回撤 | 判断是否已经从高位回调到可观察区 |
| 近20日成交量变化 | 判断缩量回调或放量转强 |
| 近60日前低 | 判断箱体下沿和双底支撑 |

## 六、筛选分层

输出分为五层。

### 1. 重点观察

满足任一买点模型，且没有触发排除条件。

重点观察不是无脑买入，而是代表：

- 位置已经接近支撑。
- 止损位可以明确。
- 如果次日或盘中转强，值得优先盯。

### 2. 触发买点

比重点观察更强，代表当日已经出现弹簧松开的迹象。

触发买点应输出明确行动说明：

- 买点类型。
- 当前价格。
- 支撑位。
- 止损位。
- 为什么今天触发。
- 若买入后几天内不对，如何处理。

### 3. 等待回调

趋势还可以，但价格离支撑太远。

常见情况包括：

- 结构本身没坏，但 `risk_to_stop_pct > 18%`，赔率已经不舒服。
- 右侧已经走出来一段，继续追的试错成本太高。

这类标的不追，只记录等待位置：

- 等 MA20。
- 等 MA40。
- 等前低或箱体下沿。
- 等第一次缩量回调。

### 4. 排除观察

已经跌破关键支撑，或进入下降通道。

排除不是永远不看，而是代表当前不适合用弹簧策略参与，需要等重新筑底、第一根启动阳线或重新站回关键均线。

### 5. 数据不足

行情数据太短、价格异常或数据源缺失。

## 七、买点模型

### 模型 A：MA40 弹簧压紧

这是第一版最重要的模型。

适用场景：

- 方向或板块已经被市场选择过。
- 标的曾经有过一段上涨，且仍处于中期趋势未破坏状态。
- 当前回调到 40 日线附近。

触发条件：

1. 趋势前提成立：
   - `ret40 > 0`
   - `MA40` 最近 5 日不下行
   - 过去 40 日至少 70% 的交易日收盘价位于 `MA40` 上方
2. 最近 3 日至少一次真正压到 `MA40` 或附近支撑：
   - `recent_3d_low <= MA40 * 1.01`
   - 同时不能明显跌穿：`recent_3d_low >= MA40 * 0.97`
   - 最新价格仍在支撑边缘：`close >= MA40 * 0.99`
3. 近 20 日最大回撤至少 5%，说明不是高位直接追涨，而是已经经历过一次像样回调。
4. 回调阶段出现缩量和收敛：
   - `vol5 <= vol20 * 0.9`
   - 最近 5 日平均振幅低于最近 20 日平均振幅的 90%
5. 最近 10 日内不能出现放量破位：
   - 例如 `close < MA40 * 0.97 且 volume >= vol20 * 1.3`
   - 或者出现 `日跌幅 <= -3% 且 volume >= vol20 * 1.3`
6. 弹簧开始松开时，不只要求“翻红”，而是要求右侧转强：
   - 价格重新站上 `MA5`
   - 同时突破前一日高点，或突破近 5 日小平台高点
   - 当天涨幅必须大于 `5%`

分层：

- 收盘模式满足 1 到 5：`重点观察`
- 收盘模式同时满足 6：`触发买点`
- 盘中模式满足 1 到 5：`盘中预警`
- 盘中模式同时满足 6：`盘中触发`

止损：

- ETF：MA40 下方 2% 到 3%。
- 个股：MA40 下方 5% 到 8%，或最近前低下方 2%。
- 若前低更近，优先使用前低作为证伪点。

### 模型 B：MA20 第一次回档

适用场景：

- 标的刚启动一段趋势。
- MA20 已经向上。
- 价格第一次回到 MA20 附近。

触发条件：

1. `MA20 > MA40`。
2. 过去 40 日内，至少 70% 的交易日收盘价在 MA40 上方。
3. 当前价格进入 MA20 上下 3%。
4. 回调过程没有放量大阴线：最近 10 日内不存在 `日跌幅 <= -3% 且成交量 >= 近20日均量 * 1.3`。
5. 当日重新站上 MA5，或收盘价高于前一日高点。

分层：

- 收盘模式满足 1 到 4：`重点观察`
- 收盘模式同时满足 5：`触发买点`
- 盘中模式满足 1 到 4：`盘中预警`
- 盘中模式同时满足 5：`盘中触发`

止损：

- MA20 下方 3% 到 5%。
- 如果 MA40 更近，可使用 MA40 作为最终防线。

### 模型 C：双底或二次压紧

适用场景：

- 第一次回调没有直接走出趋势。
- 第二次回踩前低但没有有效跌破。
- 右侧开始重新转强。

触发条件：

1. 近 60 日出现两个相近低点，且这两个低点必须是最近连续的两个有效低点，中间不能再夹一个新的低点；两个低点之间至少间隔 10 个交易日。
2. 第二低点相对第一低点在 `-1% ~ +3%` 之间，也就是允许略破，但不能明显破位。
3. 两个低点之间必须有一段像样的反弹，中间最高价相对低点反弹至少 6%，否则更像普通震荡而不是双底。
4. 两个候选低点之间如果已经出现新的更低低点，则前一个低点立即失效，必须丢掉旧左底，改用这个新的更低点重新识别结构。
5. 第二次回踩阶段的平均成交量应低于第一次下跌阶段，建议至少满足 `第二次回踩阶段均量 <= 第一次下跌阶段均量 * 0.85`。
6. 第二次回踩后的收盘价重新站上 MA5。
7. 第二次回踩后，价格还要突破右侧小平台，至少突破第二低点之后近 5 日高点。
8. 最近 10 日内不能出现放量破位：`收盘价 < 第一低点 * 0.99 且成交量 >= 近20日均量 * 1.3` 视为破坏结构。

分层：

- 收盘模式满足 1 到 5，且没有放量破位：`重点观察`
- 收盘模式同时满足 6 到 8：`触发买点`
- 盘中模式满足 1 到 5，且没有放量破位：`盘中预警`
- 盘中模式实时价重新站上 MA5，并突破右侧小平台，且没有放量破位：`盘中触发`

止损：

- 双底低点下方 2% 到 3%。

### 模型 D：整理后第一根启动阳线

这是允许追的例外模型。

适用场景：

- 标的长时间横盘或底部整理。
- 突然出现第一根放量大阳线。
- 用户给出的方向有事件催化或市场关注度正在升温。

触发条件：

1. 过去 10 日振幅收敛：`近10日最高价 / 近10日最低价 - 1 <= 15%`。
2. 当日涨幅超过 5%。
3. 当日成交量超过近 5 日均量的 1.5 倍。
4. 收盘价突破近 10 日高点。
5. 只允许在第一根启动阳线当天触发；如果前一日已经构成启动阳线，第二天即使继续上涨，也不再按该模型买入。

分层：

- 收盘模式满足全部条件：`触发买点`
- 盘中模式满足价格突破和涨幅条件，且量能进度达到对应交易时间的放量预警阈值：`盘中触发`

止损：

- 启动阳线实体被完全吞没。
- 或收盘价重新跌回近 10 日整理平台。

仓位提示：

- 该模型赔率不如支撑位附近，报告中应提示 `轻仓试错`。

## 八、排除条件

满足任一条件，直接进入 `排除观察`。

1. 收盘价跌破 MA40 超过 3%，且 MA20 已经低于 MA40。
2. 最近 20 日高低点持续下移：近 20 日后 10 日的最高价低于前 10 日最高价，且后 10 日最低价低于前 10 日最低价。
3. 放量跌破前低或箱体下沿。
4. 弹簧松开后又反复压回支撑位，超过两次仍未走强。
5. 高位累计换手过大后放量滞涨，且离 MA20 / MA40 都很远。
6. 最新价格距离最近合理止损位过远，且同时已经出现明显破位或下降通道迹象。

盘中模式下，若实时价触发 1 到 4 中任一价格证伪条件，先进入 `盘中失效`。收盘后若仍未收回关键支撑，再正式进入 `排除观察`。

## 九、评分规则

第一版采用人工可读的 100 分制，方便排序。

| 项目 | 分值 |
| --- | --- |
| 方向匹配和相对强势 | 20 |
| 价格接近 MA20 / MA40 / 前低支撑 | 25 |
| 回调缩量或波动收敛 | 15 |
| 当日重新转强 | 20 |
| 止损距离合理：`risk_to_stop_pct <= 12%` | 10 |
| 没有下降通道或破位风险 | 10 |

推荐分层：

- `80 分以上`：触发买点。
- `65 到 79 分`：重点观察。
- `45 到 64 分`：等待回调或等待确认。
- `45 分以下`：排除观察。

盘中模式的分数也是临时分数，必须额外输出 `confidence: provisional`。

收盘确认模式的分数输出 `confidence: confirmed`。

## 十一、每日解读模板

每次运行后，报告顶部应给出一句人能直接理解的话：

```text
机器人方向今日有 1 个触发买点、2 个重点观察。优先看机器人ETF是否能在 MA40 上方继续转强；汇川技术还差右侧确认，不宜提前追。埃斯顿趋势还在但离支撑偏远，等回调。
```

然后按优先级输出：

1. 今日可盯的触发买点。
2. 明日重点观察。
3. 等待回调的位置。
4. 已经排除的原因。

盘中模式的解读必须强调“临时”：

```text
机器人方向盘中有 1 个临时触发、2 个接近支撑预警。机器人ETF实时价已站回 MA5，但这是盘中信号，需要收盘确认；若午后重新跌回 MA40 下方，则不视为正式买点。
```

## 十二、第一版暂不处理的内容

为了先跑起来，第一版不做这些事情：

- 不解析新闻和事件强度。
- 不自动识别全市场主线。
- 不做盘中分钟级信号。
- 不自动计算真实持仓仓位。
- 不根据财务、研报、融资余额做复杂选股。
- 不用实时盘口挂单深度做复杂成交判断。
- 不做分钟级 K 线形态识别。

这些可以在规则稳定后逐步加入。

## 十三、后续实现建议

后续写代码时，建议拆成四个小模块：

1. `watchlist` 配置读取：读取用户给出的方向和标的池。
2. 行情加载：复用现有 `research.data_loader.load_symbol_frames()`。
3. 实时行情加载：盘中模式接入 `data.realtime_quote_source.RealtimeQuoteSource`。
4. 信号计算：独立实现 MA20 / MA40 / 双底 / 启动阳线模型，并区分盘中临时信号和收盘确认信号。
5. 报告输出：同时输出 JSON、CSV 和一段 Markdown 摘要。

这样它可以独立运行，也能在未来接入现有 `research_market_states.py` 的市场阶段判断。

建议输出字段至少包括：

| 字段 | 说明 |
| --- | --- |
| `mode` | `intraday` 或 `close_confirmed` |
| `signal_timing` | 信号来自盘中实时价还是收盘日线 |
| `confidence` | `provisional` 或 `confirmed` |
| `fetched_at` | 实时行情抓取时间 |
| `trade_time` | 行情源返回的交易时间 |
| `latest_price` | 盘中为实时价，收盘后为收盘价 |
| `needs_close_confirmation` | 盘中信号还需要确认的条件 |
